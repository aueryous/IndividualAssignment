2. Data Quality Issues, Missing Data & Anomalies

2.1 Schema / Structural Problems

Missing column header for timestamp. The first column (the datetime index) has None as its header. Any automated script or pipeline that reads this file by column name will silently skip or misprocess the timestamps.
  • Ghost empty column: Column index 19 is labelled Unnamed: 0 and contains no data in any row — a pandas export artifact. It adds noise and may confuse tools that infer column count from headers.
  • No units documented: It is never stated in the file whether temperatures are °C or °F, or whether efficiency is expressed as COP, kW/ton, or another metric. (Context strongly suggests °C and a dimensionless COP ratio capped at 2.0, but this must be assumed by the analyst.)

2.2 Zeros Used as a "No Data" Sentinel

14.5% of all rows (6,781 rows) have every sensor value equal to exactly 0. These are not valid physical readings — 0°C condenser water is implausible for a tropical/summer climate — yet the file uses 0 as a proxy for "system offline" or "sensor not reporting." This makes it impossible to distinguish a legitimate zero from a missing value without domain knowledge.

2.3 Frozen / Stale Sensor Data

The CondenserWaterReturnTemp column reports the identical value for 3,283 consecutive rows — approximately 6.8 continuous days — starting 2011-06-17 16:51. A real sensor reading physically cannot stay perfectly constant for that duration. This is a data logger or communication freeze, not a valid steady state. The same freeze affects all other columns simultaneously (they all show identical all-zero blocks), confirming a logging outage rather than a coincidental operational plateau.

2.5 Efficiency Values Hard-Capped at 2.0

All five efficiency columns share a hard ceiling of exactly 2.0. During high-performance operating periods, the true COP may exceed this value but is simply not recorded. The cap affects 33–262 readings per channel (up to 0.9% of valid readings), making peak-efficiency analysis unreliable.

2.6 Physical Consistency Flag

Some rows show TotalChiller_kW = 0 while TotalCondenserWaterPump_kW > 0. Running condenser pumps with no chiller load is unusual and likely represents startup/shutdown transition states, but the data provides no mode flag to confirm this interpretation.

2.7 Power Meter Metadata Typo

In uwc_pm_meta.xlsx, the PointType field contains both "LightingSubmeter" (22 records) and "LIghtingSubmeter" (4 records — capital I). Any GROUP BY or filter on this field will silently miss 4 meters, under-reporting lighting energy by roughly 15% of that submeter category.

3.1 Energy Efficiency KPIs

4. Key Findings Summary

What AI tools typically get right: Column identification, basic statistics, spotting NULL values, and generating generic KPI lists for chiller plants.
What AI tools typically miss:
  • Zeros-as-sentinel vs. true zeros (requires domain knowledge to distinguish)
  • The 6.8-day frozen sensor run (requires sequence analysis, not just column stats)
  • The missing timestamp header (only visible by inspecting raw cell A1)
  • The efficiency hard cap at 2.0 (appears as a valid max value without context)
  • The metadata typo LIghtingSubmeter (easy to overlook without a GROUP BY)
  • Physical inconsistency between chiller kW = 0 and condenser pump kW > 0
